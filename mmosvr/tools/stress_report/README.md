# Stress Test & Report

End-to-end load testing tools for mmosvr. Measures server tick time,
broadcast latency, zone population, and per-client RTT while 300 bots
exercise the game server.

## Files

- `run_stress.bat` — launch DummyClient in stress mode and generate HTML report.
- `generate_report.js` — standalone report generator (consumes two CSVs).

## Prerequisites

- Built `mmosvr.sln` (Debug or Release).
- `node` on PATH (for report generation).
- LoginServer and GameServer running (`mmosvr/bin/{Debug,Release}/x64/`).

## Procedure

1. Build (Release recommended for real numbers):

       msbuild mmosvr.sln /p:Configuration=Release /p:Platform=x64 /m

2. Start LoginServer and GameServer in separate consoles.
   Server writes its CSV to `mmosvr/metrics/gameserver_metrics.csv`
   (path is relative to the server's CWD, so run them from the project root).

3. Run the stress driver:

       mmosvr\tools\stress_report\run_stress.bat Release

   Or manually:

       cd mmosvr\bin\Release\x64
       DummyClient.exe --stress --bots=300 --ramp-sec=60 --hold-sec=120 ^
           --csv=..\..\..\metrics\client_metrics.csv

4. Open `mmosvr/metrics/report.html` in a browser.

## What the report shows

### Server (GameServer)

- Tick latency percentiles (p50/p95/p99/max) and tick-over-budget count.
- Zone::Update / Broadcast / PacketQueue::Flush p95 breakdown.
- Current sessions / players / monsters / projectiles.
- Packet rate and bandwidth (TX/RX).

### Client (DummyClient bots)

- Total / connected / in-game bot counts.
- End-to-end RTT for C_PlayerMove echoes.
- Login / connect / enter-game failure rates.

## CLI options

See `DummyClient.exe --help`. Common tweaks:

    --bots=N          load size (default 300)
    --ramp-sec=N      linear spawn window (default 60)
    --hold-sec=N      measurement window after ramp (default 120)
    --move-ms=N       per-bot C_PlayerMove period (default 200)
    --host=IP         non-loopback server

## Output paths

- `mmosvr/metrics/gameserver_metrics.csv` — written by server every 5s.
- `mmosvr/metrics/client_metrics.csv` — written by DummyClient every 5s.
- `mmosvr/metrics/report.html` — generated by `generate_report.js`.
